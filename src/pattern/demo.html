01 — @layer
Layer Architecture

The entire framework is wrapped in named @layer blocks. Layer order controls the cascade — styles in later layers always win, no !important needed. Add your overrides to @layer overrides.

Layer cascade (bottom wins)
resetBox sizing, margin, baseline normalisation
tokensSpace scale, color, container widths
macroGrid, containers, page shells, subgrid
microFlex utilities, container queries
patternsCluster, stack, masonry, overlay, reel…
utilitiesLogical margin/padding, display, RTL
overrides← Your styles here. Always win.

03 — Subgrid
Inherited Column Tracks

Children of .grid can opt into .subgrid to inherit the parent's column tracks. Nested elements stay column-aligned without manual math — notice how all card sections align across cards below.

Classes: .subgrid (columns) · .subgrid--rows (rows) · .subgrid--both
.grid → .col-4.subgrid-card (grid-template-rows: subgrid, grid-row: span 3) — top/title/body rows align across all cards

04 — Container Queries
Component-Responsive Layout

Add .cq to any parent to make it a container. Children can then use @container-aware classes that respond to the component's width, not the screen. The same card component adapts in a narrow column, a medium panel, or a full-width slot.

Activate: .cq on any parent → Use: .cq-flex--col@sm · .cq-flex--col@md · .cq-hide@sm · .cq-card pattern

06 — Logical Properties + RTL
Writing-Mode Agnostic Layout

Every spacing and sizing utility uses logical properties (margin-inline, padding-block, inline-size, block-size…). Flip dir="rtl" on any element and the layout mirrors automatically — no extra classes needed.

Logical margin: .mb-{n} · .mbs-{n} · .mbe-{n} · .mi-{n} · .mis-auto · .mie-auto
Logical padding: .pb-{n} · .pi-{n}
Sizing: .w-min · .w-max · .w-fit · .w-full · .h-screen
Border: .border-inline-start · .border-block-end (auto-flips in RTL)
